# eMotor-Studio 产品重定位：国产化 VESC-like 电机控制开发平台

Date: 2026-05-14

## 新定位

eMotor-Studio 的新定位是：国产化 VESC-like 电机控制开发平台。

它不是简单串口助手，不是普通 PySide Demo，也不是一次性上位机界面。它应该逐步成为“上位机 + AxDr_L 固件 + 自研硬件 + 中文工程调试流程 + 教程课程 + 数据沉淀 + AI 辅助调试”的工具链入口。

当前仍处于私有原型阶段，优先快速验证产品结构、UI 工作流、协议设计和 AxDr_L 适配方向。

## 为什么有价值

- 中文优先，降低电机控制学习和调试门槛。
- 面向国产硬件和 AxDr_L 固件，避免只围绕国外控制器生态。
- 将工程调试流程标准化：连接、配置、实时数据、采样数据、故障、日志、报告。
- 能沉淀参数、波形、故障和项目案例，服务课程、教程和社群。
- 后续可结合 AI 辅助调试，自动解释异常波形、故障码和参数风险。
- 可以作为硬件教程、控制课程和工程项目的统一入口。

## 对标对象

- VESC Tool：主要对标对象，参考其页面架构、实时数据、采样数据、参数配置、终端和固件工作流。
- SimpleFOCStudio：参考串口优先、调参和教学友好工作流。
- ODrive GUI：参考设备发现、后端隔离和电机控制配置思路。
- Spectral-motor-GUI：参考产品化电机驱动 GUI 和 UART/CAN 路线。
- PID 教学类项目：参考 Mock 仿真、波形、命令和课程演示路径。

## 与 VESC 的差异

- 面向 AxDr_L 固件和自研国产硬件，而不是 VESC 硬件生态。
- 中文优先，术语、流程和教程面向国内学生与工程师。
- 更强调课程和教学友好，保留 Mock 模式、报告和知识库路线。
- 预留 AI 辅助调试入口。
- 使用 Python + PySide6 快速迭代；未来可考虑 C++/Qt 分支或直接 fork 研究。
- 协议、参数、故障和控制对象必须与 AxDr_L 自身设计对齐。

## 短期目标

- 完成 AxDr_L 专属上位机页面结构。
- 完成 AxDr_L 串口协议草案。
- 实现 SerialBackend 最小 PING。
- 接入 firmware info、telemetry、参数读写、故障读取和基础命令。
- 保持中文 UI、Mock 模式和自动审查体系。

## 中期目标

- CAN 支持和多轴管理。
- 电流环、速度环、位置环测试与参数整定。
- 电机参数辨识。
- 固件升级和版本管理。
- 采样数据触发、波形分析和日志复盘。
- 参数模板、课程案例和硬件教程联动。

## 长期目标

- 国产 VESC-like 生态。
- 自研硬件套件和固件 SDK。
- 上位机插件系统。
- 教程平台和课程资源。
- AI 调试助手：波形解释、故障诊断、参数建议、报告生成。
- 多项目、多硬件、多固件版本的长期知识库。
